    CWT

    A consent web token represents the GDPR consents expressed by a user. It can be used for storage or to be shared with third-parties.

    Example:

    const token = new CWT(
  'issuer',
  'user@domain.com',
  'email'
);

    public addConsent(purpose: string, vendorId: string, scopeId: string) source

    Add a consent given by the user

    Params:

    NameTypeAttributeDescription
    purpose string

    The purpose for which the user has given consent
    vendorId string

    The unique vendor ID for which the user has given consent. Use * to indicate that the user has given consent for all vendors)
    scopeId string
    • optional

    The scope for which the user has given consent (example: email, website, a unique internal ID, etc.). This is used to further limit the consent to a specific context as needed. It allows doing local (one website) vs global (all websites) consents easily. Use * to indicate that the user has given consent for any scope.

    Example:

    const token = new CWT('issuer');
token.addConsent(
  CWT.Purposes.Cookies,
  'didomi',
  '*'
);

    public hasConsent(purpose: string, vendorId: string, scopeId: string): boolean source

    Check if the user has given consent for a specific purpose, vendor and scope

    If the vendor or the scope are not provided, the function ignores them and simply checks if consent has been given for the purpose. Make sure that you provide a vendor or scope to match the level of compliance you are targeting. For instance, if you want to make sure that the user has given the most specific purpose, you'll want to include both the vendor and the scope in your query.

    Params:

    NameTypeAttributeDescription
    purpose string

    Purpose
    vendorId string
    • optional

    Unique ID of the vendor to check consent for
    scopeId string
    • optional

    Scope to check consent for

    Return:

    boolean

    Example:

    const token = new CWT('issuer');
token.addConsent(CWT.Purposes.Cookies, 'didomi', '.didomi.io');
token.hasConsent(CWT.Purposes.Cookies, 'didomi');
